## プロジェクトの目的

Jicoo（オンライン予約システム）において、指定されたレッスン時間（20:25）の予約を自動的に行う BOT システムを開発する。手動での予約作業を自動化し、効率的にレッスン予約を確保することを目的とする。

## 背景と必要性

- 手動での予約作業は時間がかかり、希望の時間枠を逃す可能性がある
- 予約開始時刻に即座に対応する必要がある
- 複数の講師の URL を監視する必要がある
- ログインポップアップが表示される場合がある

## プロジェクトスコープ

### 含まれるもの

- 4 つの URL の自動監視機能
- ログインポップアップの自動処理
- 1 週間後の日付の自動選択
- 20:25 タイムスロットの監視と自動予約
- 予約フォームの自動入力
- エラーハンドリングとリトライ機能
- ログ出力機能

### 含まれないもの

- 予約のキャンセル機能
- 複数の時間枠への対応（20:25 のみ）
- 予約履歴の管理機能
- 通知機能（メール・Slack 等）

## 予算・リソース

- **開発工数**: 約 10-15 時間（見積もり）
- **技術スタック**: Java 17+, Selenium 4+, WebDriverManager
- **開発環境**: ローカル PC
- **運用環境**: ローカル PC（常時起動）

## 制約条件

- Java 17 以上が必要
- Chrome ブラウザが必要
- インターネット接続が必要
- 24 時間稼働を想定（必要に応じて）

## 前提条件

- Jicoo の Web サイト構造が変更されない
- ログイン情報（students/uluru2024）が有効である
- 予約可能な時間枠が存在する

## 成功基準

1. 4 つの URL すべてで正常に動作する
2. ログインポップアップを自動処理できる
3. 20:25 のタイムスロットを自動的に予約できる
4. エラー発生時に適切にリトライする
5. ログが適切に出力される
6. ヘッドレスモードで動作する
